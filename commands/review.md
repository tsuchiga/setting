# コードレビュー

直近で変更されたファイルを対象にコードレビューを行い、日本語で報告してください。

## 手順

1. `git diff HEAD` または `git diff --staged` で変更内容を取得
   - 変更がない場合は `git diff HEAD~1` で直前のコミットを対象にする
2. 変更されたファイルを読み込んで内容を把握する

## チェック項目

以下の観点でレビューし、問題があれば **重要度（高/中/低）** をつけて報告してください：

- **バグ・ロジックエラー**: 意図しない動作になっていないか
- **セキュリティ**: SQL injection、XSS、機密情報のハードコードなど
- **型安全性**: TypeScriptの場合、anyの乱用・型の不整合
- **パフォーマンス**: 不必要なループ・重複処理・N+1問題
- **可読性**: 関数・変数名がわかりやすいか、コードが複雑すぎないか

## 出力形式

問題点がなければ「問題なし」と明記してください。
問題点がある場合は、該当ファイル名と行番号を示して具体的に説明してください。
