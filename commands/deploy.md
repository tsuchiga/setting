# デプロイ

デプロイを実行する前に以下の手順を必ず行ってください。

## 手順

### 1. 未コミット変更の確認とコミット

`git status` を実行する。

**未コミット変更がある場合：**
- 変更ファイルの一覧を表示する
- 「未コミットの変更があります。先にコミットします。」と報告する
- 変更内容を確認し、適切なコミットメッセージを作成する
- `git add .` → `git commit -m "..."` を実行してコミットする
- コミット完了後、次のステップへ進む

**変更がない場合：**
- そのまま次のステップへ進む

### 2. テストの存在確認

テストフレームワークとテストファイルが存在するか確認する。

**テストが存在しない場合：**
- 「テストが見つかりません。/test-setup でテスト環境を整えることができます。」と案内する
- 「テストなしでデプロイを続行しますか？」とユーザーに確認を取る
- ユーザーが拒否した場合はデプロイを停止する
- ユーザーが承認した場合はステップ4へ進む

**テストが存在する場合：**
- 次のステップへ進む

### 3. 全テストの実行

以下を順番に実行する：
- Unit テスト
- Integration テスト
- E2E テスト（Playwright）

プロジェクトに応じて適切なコマンドを使用する（例：`npm test`、`npx playwright test`、`pytest` など）。

**失敗がある場合：**
- デプロイを停止する
- 失敗したテスト名・ファイル・エラー内容を詳しく報告する
- 修正はユーザーの判断に委ねる。自動修正はしない

**全テスト通過の場合：**
- 「全テストが通過しました。デプロイを実行してよいですか？」とユーザーに確認する

### 4. デプロイ実行

ユーザーの承認を得てからデプロイを実行する。
