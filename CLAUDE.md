# ユーザー設定

## プロフィール

- 役割：社内エンジニア・AIコンサルタント・CFO の掛け持ち
- 主な技術スタック：TypeScript, Python, Next.js, Remix, Prisma, Shopify, Supabase
- パッケージマネージャー：npm をメインで使用
- 環境：Windows 11

## 回答スタイル

- **必ず日本語で回答する**
- 不明点・曖昧な点は正直に伝える。推測で進めない
- 作業に必要な情報が不足している場合は、進める前に確認する
- 説明は簡潔に。冗長な前置きや要約は不要

## コーディング方針

- 新機能を実装する際は、関連する既存機能への影響を必ず確認してから実装する
- 既存機能に影響が出る可能性がある場合は、実装前にユーザーに確認する
- コードに不要なコメントを追加しない

## テスト方針

### テストの種類（全て必須）
- **Unit テスト**：関数・コンポーネント単位
- **Integration テスト**：API・DB連携
- **E2E テスト**：Playwright による画面操作テスト

### 実装時のタイミング

**設計書ベースの実装 → TDD**
1. 設計書から機能一覧を抽出する
2. テストケースを設計する（正常系・異常系・境界値）
3. テストファイルを先に作成する（この時点でテストは失敗する）
4. テストが通るように実装する
5. 全テスト通過を確認してから次の機能へ進む

**機能追加・修正 → 実装と同時**
1. 関連する既存テストを確認する
2. 実装しながらテストを追加・更新する
3. 関連する既存テストが壊れていないか確認する
4. 全テスト通過を確認する

### デプロイ時
- デプロイ依頼を受けたら、必ず全テストを実行してから行う
- テスト失敗時はデプロイを停止し、失敗原因を報告してユーザーの判断を仰ぐ
- 全テスト通過後、ユーザーに確認を取ってからデプロイを実行する

## Git 操作時の必須アナウンス

`git push` / `git pull` / `git clone` を実行する際は、必ず以下をアナウンスする：

> ⚠️ **settings.local.json は Git 管理外です。**

> 配置先：`~/.claude/settings.local.json`
> 内容：Supabase トークン・個人用権限設定
