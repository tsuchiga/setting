# 共通セキュリティルール（全言語共通）

## コミット前の必須チェック
- APIキー・パスワード・トークンのハードコードがないか確認する
- ユーザー入力のバリデーションが実装されているか確認する
- SQL インジェクション対策（パラメータ化クエリ）が実装されているか確認する
- XSS 対策（HTML のサニタイズ）が実装されているか確認する
- 認証・認可の確認が実装されているか確認する

## シークレット管理の原則
- 機密情報は環境変数または専用のシークレット管理システムを使う
- アプリケーション起動時に必要なシークレットの存在を検証する
- 漏洩が疑われる認証情報は即座にローテーションする

## セキュリティ問題発生時の対応
1. 作業を即座に停止する
2. 影響範囲を調査する
3. 露出した認証情報をすべて更新する
4. 類似の脆弱性がないかコードベース全体を確認する
