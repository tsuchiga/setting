# TypeScript コーディングスタイル

対象ファイル: `**/*.ts`, `**/*.tsx`, `**/*.js`, `**/*.jsx`

## 型定義
- エクスポートする関数には必ずパラメータと戻り値の型を明示する
- オブジェクト形状には `interface`、ユニオン・インターセクションには `type` を使う
- `enum` は原則使用禁止（相互運用性が必要な場合のみ許可）
- `any` はアプリケーションコードで使用禁止。不明な型は `unknown` を使い安全にナローイングする

## React コンポーネント
- コンポーネントの props は必ず `interface` または `type` で定義する
- コールバックは型を明示する
- `React.FC` の使用には理由が必要

## イミュータビリティ
- ミューテーションではなくスプレッド演算子で更新する
- パラメータには `Readonly<T>` を使い不変性を担保する

## エラーハンドリング
- async/await には try-catch を実装する
- catch では `unknown` 型を受け取り `instanceof Error` で安全にナローイングする

## バリデーション
- Zod を使ったスキーマベースのバリデーションを推奨する
- スキーマをバリデーションと TypeScript 型の単一の源泉とする

## ロギング
- 本番コードに `console.log` を残さない。適切なロギングライブラリを使う
